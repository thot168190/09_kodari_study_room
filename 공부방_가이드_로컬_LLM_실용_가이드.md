## ⚠️ 16GB RAM 하드웨어 한계 진단

* **OS + 기본 백그라운드 앱**: 약 4~6GB 점유
* **실제 사용 가능한 잔여 RAM**: 약 10~12GB
* **동작 원리**: 실행하는 모델과 컨텍스트 데이터가 물리 RAM 내에 통째로 적재되어야 렉 없이 작동합니다.

## 🎯 용도별 최적 로컬 모델 배정

### ⚡ 1. 초고속 일상 요약 및 한글 교정 ➔ `qwen3:4b`
* 한국어 지시문 및 스크립트 작성에 아주 매끄러운 핏을 보임.
* 짧은 질문, 요약, 문맥 변환 시 딜레이 없이 즉각 응답.
```bash
ollama run qwen3:4b "아래 문단을 3줄로 요약해줘: [내용]"
```

### ✍️ 2. 기획서 검토 및 중급 글쓰기 ➔ `gemma4:e4b`
* 블로그 초안 작성, 기획서 논리 검토, 교재 분석용.
* 단, 실행 시 약간의 딜레이(수초 대기)가 있으며, 무거운 프로그램(Final Cut, Chrome 탭 30개 등)은 잠시 닫아주시는 게 안정적입니다.

### 🌐 3. 구글 생태계 및 영문 번역 ➔ `gemma4:latest`
* 구글 Gemma 모델 고유의 논리적이고 깔끔한 영문 번역과 기술 문서 맥락 이해도가 매우 뛰어남.
