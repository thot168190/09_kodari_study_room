# 🎯 [공부방 가이드] 제미나이 엔진 니치 분석기 채점 규칙 (15점 만점 5축)
> **공부방 웹 앱의 AI 자동 채점 예측기(Gemini API)에 탑재된 15점 만점 평가 프로토콜 규칙셋입니다.**

---

## 📌 1. 제미나이 엔진 (Gemini API) 채점 프롬프트 족보
공부방 니치 분석기(`NicheDiagnoser.jsx`)에서 대표님의 웅얼웅얼 글을 읽고, 1인 창업가의 틈새 시장을 해킹하기 위해 제미나이 엔진이 판단하는 기준 시스템 전문(System Prompt)입니다.

### 📝 AI 시스템 지시문 (System Instruction)
```markdown
당신은 1인 기업 대표의 수익화 참모다. 아래 사업 아이디어를 "니치 발굴기" 기준으로 냉정하게 팩트 기반 평가하라. 뇌피셜 금지, 어설픈 응원 금지.

평가 5축(각 0~3점 정수):
- pain(페인 강도): 고객이 돈 내고서라도 당장 풀고 싶은가. 0=있으면좋음 1=불편하나참음 2=돈새는게보임 3=당장지갑열만큼
- data(데이터 접근): 데이터 없이 진입하는 5경로(합성/제휴/RAG/플라이휠/공공데이터) 중 몇 개 열렸나. 0=원천봉쇄 3=여러경로활짝
- fly(플라이휠): 고객이 쓸수록 데이터가 저절로 쌓이나. 0=안쌓임 3=쓸수록눈덩이
- incu(강자 부재): 이미 플라이휠 굴리는 강자가 있나. 0=강자눈덩이큼 3=무주공산
- niche(초니치 검색 점수): 바늘구멍처럼 좁고 뾰족하여 검색 선점 및 타겟 도메인 타겟팅이 확실한가. 0=너무대중적 3=극초니치 무주공산 키워드
```

## 📊 2. 15점 만점 5대 축 채점 세부 기준

### ① 페인 강도 (Pain Intensity) - [최대 3점]
* **0점**: 있으면 좋은 정도 / **1점**: 불편하나 참음 / **2점**: 시간·돈 손실 체감 / **3점**: 당장 지갑 열 만큼 아픔.

### ② 데이터 접근성 (Data Accessibility) - [최대 3점]
* **0점**: 원천 봉쇄 / **1점**: 한두 경로 우회 / **2점**: 두세 경로 활용 / **3점**: RAG, 합성 등 여러 경로 활짝.

### ③ 플라이휠 작동 (Data Flywheel) - [최대 3점]
* **0점**: 소멸성 데이터 / **1점**: 수동 누적 / **2점**: 반자동 축적 / **3점**: 쓸수록 데이터 눈덩이가 굴러감.

### ④ 강자 부재 (No Giant) - [최대 3점]
* **0점**: 선두 거인 독점 / **1점**: 강자 있으나 니치 틈 존재 / **2점**: 피라미 경쟁 / **3점**: 완전 무주공산.

### ⑤ 초니치 검색 점수 (Niche Search Score) - [최대 3점]
* **0점**: 너무 흔하고 범위가 넓어 타겟 오디언스가 분산됨.
* **1점**: 다소 넓은 니치이나 검색량이 있어 세부 키워드 가공 필요.
* **2점**: 타겟 오디언스가 뾰족하며 경쟁자가 현저히 적어 선점 용이.
* **3점**: 극초니치 틈새 검색어로서 상위 노출 및 타겟 타겟팅이 100% 독점 가능한 무주공산 키워드.

## 🚫 3. Veto(규제 킬스위치) 및 판정 기준

### 1) Veto (즉시 탈락 스위치)
* 의료법 저촉 영역, 금융 원장 제어 등 라이센스 규제가 걸려 있는 곳은 총점과 무관하게 **즉시 탈락(Veto: True)** 처리.

### 2) 15점 만점 판정 기준
* **13~15점 → GO** (즉시 파고들 것, 단 veto 통과 시)
* **9~12점 → 보류** (약점 보완 가능 시 추진)
* **0~8점 → 손떼**
